## Krypton3

![grafik](https://user-images.githubusercontent.com/84674087/142392351-7456c62a-eb36-4fd7-8e9a-3598043db4bc.png)

#### Solution
- The cipher text: `OMQEMDUEQMEK`
- The encryption algorithm is based upon Caesar Cipher
- We don't have the key but we have the encryption tool and we can feed it with any plain text !
- This being said:
    - `mkdir /tmp/pato`
    - `echo "abc" >> /tmp/pato/code`
    - `cd /tmp/pato`
    - `ln -s /krypton/krypton2/keyfile.dat`
    - `chmod 777 .`: the encrypt binary runs setuid krypton3, you also need to give krypton3 access to your working directory
    - `krypton/krypton2/encrypt code` --> Output: **MNO**
    - There is a 12 letter shift. Now we have to take the cipher text and reverse the 12 letter shift
    - `echo "OMQEMDUEQMEK" | tr M-ZA-L A-Z`
- **PASSWORD:** CAESARISEASY

![grafik](https://user-images.githubusercontent.com/84674087/142470763-3b8cecf9-5bdf-4c3d-aca1-e0140c0d3c49.png)

#### Solution
- The cipher text: `KSVVW BGSJD SVSIS VXBMN YQUUK BNWCU ANMJS`
- Hint 1: `Some letters are more prevalent in English than others.`
- Hint 2: `"Frequency Analysis" is your friend.`
- Hint 3: `The message plaintexts are in English`

- A small bash script to show the letter frequency of an input file:
```
#!/bin/bash

read -p 'file path: ' file

echo -e "\n"
echo -e "Entered file path: $file"
for x in {a..z}
do
        count=$(cat $file | grep -i -o $x | wc -l)
        echo "$count: $x" >> /tmp/pato/buffer
done

cat buffer | sort -n -r
rm buffer
echo -e "\n"
```

- However, it's recommended to use one of the existing online sources
- One way to perform frequency analysis is [here](https://math.dartmouth.edu/~awilson/tools/frequency_analysis.html)
- A helpful cheat sheet is [here](https://www3.nd.edu/~busiforc/handouts/cryptography/cryptography%20hints.html)
